## Insights Extraídos

### 1. Pedro Vendeu Projeto de R$2.4M com Squad
Marcos (time do Pedro) usou Squad Creator para criar proposta de governo de R$2.4M + outro de R$600K (vacinação). Pedro não precisou intervir — processo validado + Squad geraram a proposta autonomamente. Prova concreta de que squads validados entregam resultado sem o founder.

### 2. BBB = Case de Complexidade Real
Projeto BBB: 848 criativos, 424 cortes, 4 meses de entrega. Análise em tempo real de inserções de mercado no programa → gera cortes → react em 24h. Todo o processo definido antecipadamente. Pedro afirma: "perto do que a gente tá fazendo é moleza."

### 3. Alan Criou Squad de Editais em 8h
Mini-Squad de governo que analisa editais abertos, pontua elegibilidade, gera teses a partir de problemas listados. Fluxo reverso: problema → tese → proposta. Inspirado em sistemas europeus. Alan ficou 2 dias sem dormir construindo isso.

### 4. Value Ladder Definida (Thiago)
Comparação com modelo Jordan Lee (4 tiers):
- **Tier 1:** R$997/mês — self-service, squads prontos, dashboard, sem Squad Creator
- **Tier 2:** Implementação guiada (2 semanas), ticket intermediário
- **Tier 3:** R$30-40K — 5 semanas com pessoa dedicada implementando, inclui Squad Creator
- Squad Creator Enterprise = Tier 3 (high-ticket implementation)

### 5. Pedro: Registries São Obrigatórios
Arquitetura Syntra usa 4 registros: Service Registry, Tool Registry, Squad Registry, Process Registry. Todo novo processo consulta o que já existe (incremental). Agentes não podem criar ferramentas novas — são forçados a usar as registradas. Isso elimina "criações aleatórias".

### 6. Syntra = IP Protegido (Sem Livro)
Pedro não publicará livro sobre Syntra porque qualquer pessoa poderia replicar a metodologia. Modelo de monetização: parcerias por nicho. Allfluence = primeiro case (nicho criativo). Depois: vendas, jurídico, imobiliário, etc. Dois paths: (1) implementação exclusiva = caro + equity; (2) licenciamento = preço normal + produtização.

### 7. Thiago Demonstrou 3-Bucket Architecture
Live demo para Alan e Pedro: content routing automático (external/business/personal) com confidence score. Entity canonicalization evita duplicatas. Clone de colaboradores alimentado automaticamente. Conteúdo bruto preservado para rastreabilidade até a fonte.

### 8. Mapeamento de Processos Bilhon = Revelador
Thiago rodou mapeamento com equipe via formulários: função, processos diários, ferramentas, frequência, tempo. Descobertas: "50 coisas manuais por dia gastando 3-4 horas." Sistema já sugere consolidação de ferramentas e identifica APIs disponíveis para automação.

### 9. Workspace Governance (Pedro)
Regra: squads NÃO podem criar novos caminhos no workspace. Devem reconhecer estrutura existente. Cada domínio precisa de um C-level representante. Domínio pessoal = clone do próprio founder. Isso previne filesystem sprawl.

### 10. Alan Editou Paths Manualmente — Erro
Alan renomeou pastas manualmente (de kebab-case para AIOX naming). Pedro alertou: "não tá doido? Todos os scripts estão apontando para os paths antigos." Lição: nunca alterar paths manualmente quando há scripts dependentes.

### 11. Narrativa do Evento Precisa de Dono
Pedro perguntou "tem alguém responsável pela narrativa da apresentação?" — ninguém tinha assumido. Sugestão: dar para Adriano (palestrante/mentor de palestrantes) como teste. Se não entregar, pelo menos sai do zero.
