## Insights Extraídos

### 1. Dashboard AIOX Reorganizado
Rafael focou em fazer tudo funcionar para o evento. Reduziu itens de menu, agrupou views. Estrutura: workspace (dados canônicos), docs (épicos/stories/documentação), artifacts (outputs por squad), apps (soluções AIOX Engine). Dashboard com widgets plugáveis e views customizáveis por usuário.

### 2. MCP Context Bloat = Problema Real
Rafael descobriu que muitos MCPs inflam a sessão e consomem contexto. Solução: converter data fetchers de MCP para serviços background (crons + API). IA só toca nos dados para análise/insights, não para coleta. Alan confirma: CEO do Perplexity disse que vão parar de usar MCPs, migrar para SQS e SLE.

### 3. Squad Marketplace Conceituado
Alan mostrou marketplace existente (de outro repo). Usuário navega, compra squad, sistema auto-instala. Rafael: "comprou, voltou no chat, já está disponível." Venda por workflow (resolver problema) ou por squad (contratar agente). Interface descreve problema → sugere squad que resolve.

### 4. Thiago: 3-Bucket Architecture Demo
Demonstração ao vivo para Rafael: external (cursos, podcasts), business (calls, reuniões), personal (reflexões). Entity canonicalization evita duplicatas. Cada bucket com RAG semântico isolado. Conteúdo bruto preservado para rastreabilidade. Auto-classificação por confidence score.

### 5. Erica: Suite Completa de Geração Visual
- **DOA Studio:** Puxa fotos + cores do brand → gera conteúdo visual consistente
- **Carrossel Generator:** Cria carrosséis Instagram alinhados com brand
- **Video Cutter Squad:** Corta lives de 4-5h em momentos-chave
- **Capas Netflix:** Thumbnails YouTube + capas de módulos estilo Netflix
- Tudo conectado com identidade visual da empresa via pasta brand no workspace

### 6. Rafael: Pipeline Completo de Lives
Fluxo end-to-end: ideia → título → thumbnail → descrição YouTube → agenda → convite WhatsApp → slides (via design system) → transmissão → download automático → Google Drive → transcrição com timestamps → cortes → aprovação/recusa (com motivos) → slots de publicação → sincronização com time de tráfego. Gerenciado via ClickUp.

### 7. Custo API Descontrolou: R$30K
Alan gastou ~R$30K em Azure (Opus 4.6) achando que era crédito gratuito. Descobriu que Azure não cobre Opus 4.6. Precisa colocar limites de custo. Lição: monitorar custos de API antes de rodar heavy workloads.

### 8. Shared Repo Criado
Alan criou repositório "dashboard-premium" no GitHub da SYNCRA. Adicionou Thiago e Rafael. Todos fazem push direto na main. Design system será compartilhado ali. Erica vai subir suas tools também.

### 9. 3D World como View Integrada
Thiago tem versão 3D (Spline) dos agentes trabalhando em workflows. Decisão: World não deve ser item separado no menu — deve ser view integrada no dashboard. Quando agentes trabalham num workflow, o usuário vê a mesma interação no mapa 3D.

### 10. Multi-tool Challenge
Empresas usam ClickUp, Asana, Trello, Jira. Dashboard precisa abstrair isso. Por enquanto: demonstrar dentro do dashboard usando ClickUp como referência. Depois modularizar com MCPs/APIs por ferramenta.

## Tensões Identificadas

1. **Custo de API sem controle:** R$30K gastos sem perceber. Sem limites de spend configurados.
2. **Muitos contribuidores, pouca coordenação:** Cada um criando features isoladamente. Repo compartilhado ajuda mas sem project board ainda.
3. **Multi-tool fragmentation:** ClickUp, Asana, Trello — cada empresa usa algo diferente. Dashboard precisa abstrair mas sem solução definida.
4. **Evento como deadline forçado:** "O evento foi muito do nada" — correria para entregar algo funcional sem planejamento adequado.
5. **MCP overhead:** MCPs consomem muita janela de contexto. Migração para serviços é consenso mas não implementada ainda.
