## Insights Extraídos

### 1. Workspace = Core IP do AIOX
O workspace (fonte da verdade dos dados do negócio) é o diferencial competitivo principal. Contém taxonomias, templates por área, context engineering, e integração multi-negócio. Alan investiu "100M+ tokens e muitas semanas" na construção. Não deve ser exposto em código aberto.

### 2. Context Engineering é o Multiplicador
Script de inicialização carrega contexto dinâmico (empresa, produto, campanha, status, referências). Agente inicia PRONTO — com 1 comando avança. Otimizado para economia de tokens. Isso é o que faz os agentes produzirem output de qualidade sem prompts manuais.

### 3. Taxonomy-First Design
Templates são criados APÓS estudo de taxonomias (contexto, entidades, glossário, workflows). Sem taxonomia → templates frágeis. Modelagem de dados precede criação de estruturas.

### 4. Tier Strategy Definida
- **Community:** Squad Creator base (free, open source) — já é "muito forte"
- **Pro:** Squad Creator Pro + dados via banco de dados (sem workspace local)
- **Enterprise:** Workspace completo + Synapse + implementação high-ticket

### 5. Black Box como Modelo de Entrega
Agentes Enterprise funcionam como Black Box: recebem input, processam internamente (com acesso ao workspace), retornam apenas output. O cliente não vê a estrutura interna.

### 6. SOP Certification Pipeline
Alan demonstrou: Squad Creator cria SOP → agente avalia (nota) → recria melhorada → certifica com ID → converte para formato máquina. Conecta com checklists automáticos.

### 7. Sandbox Per-Agent (Pedro)
Cada agente pode ter ambiente isolado (micro sandbox). Entra em loop internamente, só retorna output final. Testado com gerador de propostas — resultado positivo.

### 8. Deploy em 3 Paths (Pedro)
Testando: (1) Node.js online, (2) Claude Code Container, (3) OpenClaw/WhatsApp. Nenhum MVP definido. Objetivo: Squad como serviço na nuvem.
