---

## Insights Extraídos

### 1. Processo de Instalação Não Padronizado
Cada membro do time instalou o Squad Creator de forma diferente. José Amorim completou o processo com sucesso, mas sem documentação padrão. Gerou ruído: alguns subiram Skills pelo terminal, outros pelo sistema, sem convergência. Alan assume ownership da padronização.

### 2. WhatsApp = Canal Primário de Entrega
O Squad Creator é entregue para mentores/alunos via WhatsApp. Instalação envolve Skills + comandos de terminal. Adriano testou com WhatsApp e Telegram. Fran testou dentro do próprio WhatsApp. Funciona, mas o processo precisa ser replicável.

### 3. Reordenação do Evento: Copy → Content → Traffic
Ordem original era Traffic → Content → Copy. Mudou para Copy primeiro porque Copy faz pesquisa, análise de concorrência, gera hooks e ganchos — informações que Content consome para criar materiais. Traffic vem por último consumindo os assets dos dois anteriores.

### 4. Proteção Enterprise via Decoy Paths
Estratégia de proteção de IP: manter caminhos de arquivos no notebook que não levam a lugar nenhum ("bomba de fumaça"). O sistema conta com esses caminhos mas eles são inertes para quem não tem o modo Enterprise ativo. Quem copiar o código encontra referências que não funcionam.

### 5. Ownership de Processo é Crítico
Alan esperava que Talles fosse o dono do processo de instalação (por ter feito o primeiro sistema). Sem ownership definido, cada um improvisou. Lição: definir dono do processo antes de distribuir a tarefa.

### 6. Pressão de Timeline
"A gente tem um final de semana. Não temos mais uma semana — até segunda-feira precisa estar tudo redondo." Timeline agressivo para preparar evento.

### 7. Natalia com Fluxo de Lives
Natalia Tanaka já tem fluxo próprio para lives: briefing → roteiro → slides → thumbnail. Potencial para integrar com Squad Creator como caso de uso para curadoria de conteúdo.

### 8. Squad de Copy Gera Inputs para Todos
Copy faz: pesquisa de mercado, análise de concorrência, extração de ganchos/hooks, headlines. Esses outputs alimentam Content (criação de peças) e Traffic (criativos para anúncios). É o squad "gerador de matéria-prima" do funil.
