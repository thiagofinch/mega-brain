## EXECUTIVE SUMMARY

Este dossier consolida o conhecimento sobre Business Operating Systems - frameworks para
escalar empresas através de sistemas, playbooks, scorecards e meeting rhythms que permitem
ao fundador "sair do dia-a-dia" sem perder controle.

**Fonte Primária:** THE SCALABLE COMPANY (Ryan Deiss) - 95%
**Fontes Complementares:** Alex Hormozi (5%) - transição CEO→Owner

## FILOSOFIAS CORE

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    FILOSOFIAS DO OPERATING SYSTEM                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. "Manual scorecards > Automated scorecards"                              │
│      → Greater insights + Greater ownership                                 │
│      → Automação divorcia time dos números                                  │
│      → Preencher = forçar reflexão                                          │
│                                                                             │
│  2. "Document what moves the needle, not everything"                        │
│      → Empresas documentam tudo, usam nada                                  │
│      → Foco em power stages primeiro                                        │
│      → 2-3 ações críticas por value engine                                  │
│                                                                             │
│  3. "Annual plans = a joke"                                                 │
│      → Muito longo para prever com precisão                                 │
│      → Muito curto para ser significativo                                   │
│      → Use 12Q (3 anos) em sprints trimestrais                              │
│                                                                             │
│  4. "Right answers got you here; right QUESTIONS get you there"             │
│      → Líderes perguntam, não respondem                                     │
│      → Quem faz perguntas controla conversa                                 │
│                                                                             │
│  5. "Cross-training has ZERO downside and UNLIMITED upside"                 │
│      → Treinar todos no playbook inteiro                                    │
│      → Não só o passo que a pessoa executa                                  │
│      → Redundância = antifragilidade                                        │
│                                                                             │
│  6. "Black boxing information = fireable offense"                           │
│      → Segurar conhecimento é tóxico                                        │
│      → Documentar e compartilhar é valor core                               │
│      → "Gollum's Precious" é anti-pattern                                   │
│                                                                             │
│  7. "Playbook owner should NOT be the CEO"                                  │
│      → Delegar ownership para operadores                                    │
│      → CEO = sponsor, não executor                                          │
│                                                                             │
│  8. "Too many ad hoc meetings = planning failure signal"                    │
│      → Se o ritmo está certo, ad hocs são raros                             │
│      → Excesso de reuniões = sistema quebrado                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## MODELOS MENTAIS

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    MODELOS MENTAIS DE OPERATING SYSTEMS                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. MATRIX NUMBERS                                                          │
│     Quando você preenche números manualmente, desenvolve a habilidade       │
│     de VER padrões nos dados como Neo vê a Matrix. Automação cega.         │
│                                                                             │
│  2. ELEPHANT EATING                                                         │
│     Como comer um elefante? Uma mordida por vez.                           │
│     3 anos = 12 quarters = 12 mordidas gerenciáveis.                       │
│                                                                             │
│  3. COCA-COLA VAULT                                                         │
│     Se um processo é tão "secreto" que só uma pessoa sabe,                 │
│     você tem um ponto de falha catastrófico.                               │
│                                                                             │
│  4. GOLLUM'S PRECIOUS                                                       │
│     Funcionário que guarda conhecimento como Gollum guarda anel.           │
│     Anti-pattern. Comportamento de demissão.                               │
│                                                                             │
│  5. TRAFFIC LIGHT ZONES                                                     │
│     RED = Faça POR eles (novo, crítico)                                    │
│     YELLOW = Faça COM eles (aprendendo)                                    │
│     GREEN = ELES fazem (competentes)                                       │
│     Objetivo: Mover tudo para GREEN.                                       │
│                                                                             │
│  6. INTERRUPTION TAX                                                        │
│     30-45 minutos para recuperar concentração após interrupção.            │
│     Reuniões ad hoc cobram esse imposto repetidamente.                     │
│                                                                             │
│  7. CREATOR BLIND SPOT                                                      │
│     9/10 vezes o criador do checklist esquece passos próprios.             │
│     Field test com não-criador é OBRIGATÓRIO.                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

## FRAMEWORKS CORE

### Framework 1: ScalableOS 3-Phase Program

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         ScalableOS 3-PHASE                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐                 │
│  │    BUILD    │ ──▶  │   INSTALL   │ ──▶  │  OPTIMIZE   │                 │
│  │   6-8 sem   │      │   2-3 sem   │      │   Ongoing   │                 │
│  └─────────────┘      └─────────────┘      └─────────────┘                 │
│                                                                             │
│  BUILD (6-8 semanas):                                                       │
│  • Map value engines (Growth, Fulfillment, Innovation)                     │
│  • Identify power stages (2-3 per engine)                                  │
│  • Create playbooks for power stages                                       │
│  • Build scorecards (manual, not automated)                                │
│                                                                             │
│  INSTALL (2-3 semanas):                                                    │
│  • Train team on playbooks                                                 │
│  • Implement meeting rhythm                                                │
│  • Deploy scorecards                                                       │
│  • Assign playbook owners                                                  │
│                                                                             │
│  OPTIMIZE (ongoing):                                                       │
│  • Weekly scorecard reviews                                                │
│  • Quarterly QSP meetings                                                  │
│  • Playbook version updates                                                │
│  • Traffic light delegation progress                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Framework 3: Traffic Light Leadership

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     TRAFFIC LIGHT LEADERSHIP                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────┐                                                               │
│  │   RED   │  Do FOR them                                                  │
│  │         │  • Você executa, eles observam                                │
│  │         │  • Quando: novo processo, nova pessoa, alta criticidade       │
│  └────┬────┘                                                               │
│       │                                                                     │
│       ▼                                                                     │
│  ┌─────────┐                                                               │
│  │ YELLOW  │  Do WITH them                                                 │
│  │         │  • Execução conjunta, coaching ativo                          │
│  │         │  • Quando: processo aprendido, competência em desenvolvimento │
│  └────┬────┘                                                               │
│       │                                                                     │
│       ▼                                                                     │
│  ┌─────────┐                                                               │
│  │  GREEN  │  THEY do (you're available)                                   │
│  │         │  • Eles executam, você disponível                             │
│  │         │  • Quando: competência demonstrada, confiança alta            │
│  └─────────┘                                                               │
│                                                                             │
│  REGRA: Objetivo é mover TUDO para GREEN.                                  │
│         Se pessoa sai, volta para RED.                                     │

### Framework 4: QSP Meeting (5 Steps)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    QSP MEETING (5 STEPS)                                    │
│                    Quarterly Sprint Planning                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────────────────────────────────────────────────────────────┐      │
│  │ STEP 1: FINANCIALS (30-60 min)                                   │      │
│  │ → Review dos números do quarter anterior                         │      │
│  │ → O que funcionou? O que não funcionou?                          │      │
│  └──────────────────────────────────────────────────────────────────┘      │
│                           │                                                 │
│                           ▼                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐      │
│  │ STEP 2: NORTH STAR (60-90 min)                                   │      │
│  │ → Definir target do próximo quarter                              │      │
│  │ → 10-20% stretch over current baseline                           │      │
│  └──────────────────────────────────────────────────────────────────┘      │
│                           │                                                 │
│                           ▼                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐      │
│  │ STEP 3: INITIATIVES (2-3 horas)                                  │      │
│  │ → Identificar iniciativas para atingir target                    │      │
│  │ → Priorizar por impacto e viabilidade                            │      │
│  └──────────────────────────────────────────────────────────────────┘      │
│                           │                                                 │
│                           ▼                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐      │
│  │ STEP 4: THEME (30 min)                                           │      │
│  │ → Definir tema inspirador do quarter                             │      │
│  │ → Algo que energize o time                                       │      │
│  └──────────────────────────────────────────────────────────────────┘      │
│                           │                                                 │
│                           ▼                                                 │
│  ┌──────────────────────────────────────────────────────────────────┐      │
│  │ STEP 5: OS REVIEW (1-2 horas)                                    │      │
│  │ → Atualizar playbooks, scorecards, estrutura                     │      │
│  │ → O que precisa ser criado/revisado?                             │      │
│  └──────────────────────────────────────────────────────────────────┘      │
│                                                                             │
│  FORMATO: 2 dias presencial OU 2-3 meio-dias virtual                       │
│  FREQUÊNCIA: Trimestral (end of quarter)                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

│     ├── Visualização de estrutura de time                                  │
