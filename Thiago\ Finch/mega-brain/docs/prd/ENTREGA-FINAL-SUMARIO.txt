📈 MÉTRICAS KPI (6 PRINCIPAIS)
════════════════════════════════════════════════════════════════════════════════

1. TVR (Total Vendor Revenue)
   └─ Fórmula: SUM(revenue de todos marketplaces)
   └─ Origem: APIs de pedidos (status = completed)
   └─ Update: 5 minutos | Real-time: novo pedido
   └─ Teste AC: Soma bate com APIs ±0.1%

2. NM% (Net Margin %)
   └─ Fórmula: (TVR - Comissões) / TVR * 100
   └─ Thresholds: Verde >20%, Amber 15-20%, Vermelho <15%
   └─ Alert: Dispara quando < 15%
   └─ Teste AC: Bate com spreadsheet auditoria (50 pedidos)

3. CP (Comissões Pagas)
   └─ Fórmula: SUM(comissão de cada marketplace)
   └─ Breakdown por marketplace (ML 16.5%, TikTok 5%, Shopee 5.3%, Amazon 8.5%, Magalu 5%)
   └─ Alert: Se > 25% em um marketplace
   └─ Teste AC: Soma dos marketplaces = total ±0.01

4. SL24 (Sales Last 24h)
   └─ Fórmula: COUNT(pedidos completados últimas 24h)
   └─ Trending: vs 24h anterior (% change)
   └─ Update: Real-time quando novo pedido entra
   └─ Teste AC: Contagem exata per API

5. AOV (Average Order Value)
   └─ Fórmula: TVR / SL24
   └─ Breakdown por marketplace
   └─ Trending: vs 7d média
   └─ Teste AC: Bata com TVR / count manual

6. RR% (Return Rate)
   └─ Fórmula: COUNT(devoluções) / COUNT(pedidos) * 100
   └─ Thresholds: Verde <2%, Amber 2-5%, Vermelho >5%
   └─ Trending: 7d SMA
   └─ Teste AC: Contagem correta devoluções per API

FRONTEND:

- State: Redux ou Zustand
- Real-time: Socket.io (WebSocket)
