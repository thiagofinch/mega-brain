### 3D Playbook Process (Define → Design → Deploy)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    3D PLAYBOOK PROCESS                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  DEFINE → DESIGN → DEPLOY (sequence is critical)                   │
│                                                                     │
│  DESIGN PHASE (Do-Document-Do Again-Field Test):                   │
│  1. DO the task (playbook owner)                                   │
│  2. DOCUMENT while doing OR immediately after                      │
│  3. DO AGAIN following your own checklist                          │
│     → 9/10 times discover missing steps                            │
│  4. FIELD TEST with never-done-before person                       │
│     → "Testing DOCUMENTATION, not the person"                      │
│                                                                     │
│  DEPLOY PHASE (Triad):                                             │
│  • TRAINING: Cross-train everyone on entire playbook               │
│  • AVAILABILITY: Digital, not binders (where docs go to DIE)       │
│  • ACCOUNTABILITY: "Do we have a playbook for this?"               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────────────────┐
